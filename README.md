# Alpha Yield: EIP-712 Lazy Minting & Staking Ecosystem

A highly secure, gas-optimized decentralized application architecture built with **Foundry** and **Solidity ^0.8.20**. This ecosystem allows users to lazily mint NFTs via off-chain cryptographic signatures, stake them in a secure vault, and continuously earn ERC-20 yield via an O(1) mathematical distribution engine.

## đźŹ— Architecture Overview

The system is separated into three distinct, composable smart contracts:

1. **`LazyNFT.sol`**: An ERC-721 token that shifts the gas cost of minting entirely to the end-user. It utilizes **EIP-712 Typed Structured Data Hashing** to verify off-chain signatures generated by an authorized backend server, preventing unauthorized mints while supporting custom, dynamic metadata URIs per token.
2. **`RewardToken.sol`**: An ERC-20 utility token with a strictly controlled supply. It features an immutable access control modifier (`onlyStakingContract`) that delegates 100% of its minting capabilities to the mathematical engine of the Staking contract.
3. **`NFTStaker.sol`**: The decentralized escrow and yield engine. It securely locks user NFTs using the `IERC721Receiver` standard and calculates continuous, per-second token emissions using time-delta mathematics rather than expensive on-chain loops.

## đź”’ Core Security Implementations

* **Cryptographic Replay Protection:** The `LazyNFT` contract tracks a `usedVouchers` mapping to guarantee that a valid backend signature can only be successfully executed a single time.
* **Signature Verification (`ecrecover`):** Prevents malicious actors from forging vouchers by mathematically extracting the signer's public address from the EIP-712 payload and comparing it to an immutable, authorized backend wallet.
* **Checks-Effects-Interactions (CEI):** Strictly adhered to across all contracts. All state variables (balances, mappings, trackers) are updated *before* any external token transfers or mints occur.
* **Reentrancy Protection:** The staking engine utilizes OpenZeppelin's `ReentrancyGuard` (`nonReentrant`) on all deposit, withdrawal, and claim functions to prevent malicious cross-contract hijacking.
* **Strict Access Control:** The `RewardToken` forces a one-time-only authorization lock. Once the Staking Contract address is initialized, it cannot be overridden, guaranteeing trustless token inflation.

## đź§® The O(1) Yield Engine

Traditional staking contracts often fail due to astronomical gas costs caused by iterating over arrays to distribute rewards. This protocol utilizes **Lazy Accumulation (Time-Delta Math)** to keep gas costs flat and perfectly predictable regardless of the number of users or the duration of the stake.

**The Math Logic:**

1. **Global High-Water Mark:** The contract tracks a global `rewardPerTokenStored` variable that only moves up based on the formula: `(Time Elapsed * Reward Rate) / Total Staked`.
2. **Personal Snapshots:** When a user interacts with the protocol, an `updateReward` modifier takes a snapshot of the global high-water mark and saves it to their personal `userRewardPerTokenPaid` mapping.
3. **Gasless Accumulation:** No on-chain computation occurs while the user waits.
4. **Delta Settlement:** Upon withdrawal, the contract calculates the user's yield strictly based on the mathematical delta between the current global high-water mark and their personal snapshot.
